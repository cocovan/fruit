Fruit

A Fluid Rich User Interface Toolkit - all you need to build a rich, responsive and good looking user interface.

Introduction

Fruit is designed to be the basis of all products that have some sort of user interaction. The guidelines (that have yet to be completed) will give details on how to lay out a user interface in the best way that should be applicable across all different platforms (web sites, web applications, desktop tools, etc).

If it the product you are building is web based, then Fruit also supplies the basic technologies and styles that you would need to build your web site or web application.

Inspiration

As part of our investigation into a UI toolkit, we drew inspiration from some of the big names and leaders in UI design and user experience. Below are some examples of the products and designs that have inspired us.

Technologies Used

Bootstrap

This is an open source project from the developers are Twitter that, through standard CSS classes, allows us to quickly lay out web pages in a responsive grid. It also comes with default styles and gives web pages a "richer" feel as it comes with a lightweight set of JQuery plugins. Fruit will use Bootstrap to layout and style basic items on an HTML page.

It is heavily documented over on GitHub so go there to find out how to style your application. Eventually, the Bootstrap CSS will be overridden by our own specific styles.

Form Builder

There's a free utility on the web that allows you to design Bootstrap style forms through a drag and drop interface. It'll generate all the correct markup for your form and use the correct Bootstrap CSS classes. A word of warning - the last time I used it, the generated source code didn't quite render quite as I was expecting it to but it was very minor. If nothing else, it'll be a quick way of getting 80% there.

JQuery

JQuery is, pretty much, the industry standard JavaScript library for DOM manipulation. It is also required by Bootstrap to run some of it's more complex controls. Again, the documentation is pretty good and there is a huge community using JQuery so the chances are, anything you want to do has already been done in some form or another.

Zepto

Zepto is very similar to JQuery but doesn't contain any of the clever fallbacks to make code work on older browsers and so is suited to use with mobile sites. If you wish to load a smaller library for your mobile site, there is an fruit-mobile.min.js file that you can include instead. Please note that note all functionality from JQuery is in Zepto so make sure you read the documentation - you can't just swap them in and out.

JQueryUI

JQueryUI offers a set of themeable widgets built on top of JQuery to use on a web page to give a rich UI. We are using this alongside Bootstrap to give us a complete as possible set of control to use on the web. Where a control type appears in both Bootstrap and JQueryUI, we are using the JQueryUI version. Again, this is a very active community with lots of examples to look through.

jsTree

A JQuery plugin that gives us a tree control as neither Bootstrap or JQuery UI have one. It's pretty powerful and can load data from multiple types of backend service.

Backbone

Backbone is a JavaScript (sort of) MVC. It allows us to give structure to client side applications as well as giving us bindings and an event modal. It also includes a RESTful interface to talk to the back end.

Backbone Model Binder

As flexible as Backbone is, it doesn't natively refresh the UI when the data model is changed behind the scenes. To rectify this, we've included a plugin for Backbone that gives is this binding. The documentation on GitHub has all the information you should need.

Modernizr

Gives HTML5 features to older browsers and also allows us to load resources based on browser features or media queries.

Grunt

Front end build system built on top of Node.js. It validates JS and CSS as well as compiling LESS files. It is also used to concatenate and minifies CSS and JS files.

Grunt Contrib

Grunt is designed to be easily extensible via custom Grunt tasks. The guys at grunt-contrib have tried to put together some common tasks in one package which we're using to complete our build system.

How to Build

You don't have to build this, just grab the latest version from the releases folder or include it as a dependency from the Maven repo. If you do want to build it, then you'll need to check it out from SVN. Once you have it checked out, follow the folling steps to build Fruit:

  1. Install Node.js
  2. Install Grunt:
    • Run, from the command line, npm install -g grunt
    • npm is the Node package manager and the -g flag means install it globally
  3. Navigate to the folder where you checkedout fruit to
  4. Install grunt-contrib:
    • Run, from the command line, npm install grunt-contrib
    • No -g flag as this module is just installed locally
    • The downloaded files will also go into a folder called node_modules
  5. Run, grunt.cmd:
    • add a -v flag to see the verbose information

The build will be in the target/fruit folder and that folder will also be zipped up, with the version, in the root of the target folder. Set up an Apache alias to here to run it through Apache Web Server or it should run locally.

How to Deploy

If you're just reading this file off of disk, then it's probably not styled and doesn't look very good. You're probably wondering what's the point of this at all! Due to the dynamic resource loading of Modernizr, this page needs to be run through a web server of sort. Download and install Apache Web Server and set up an alias to the root of this folder and you should see this in all its glory.

Now that's working, you can look at the source of the this page to see how the script loading works and also the example-application.js in the js folder. Use the folder structure that's supplied and add in your application specific files to the correct location.
